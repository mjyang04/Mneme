# Mneme CoreML e5 Runtime Path

**Goal:** Add the documented multilingual-e5 CoreML path without breaking the fully local runtime invariant.

## Scope

Implemented in this pass:
- Local e5 preprocessing support: `query:` / `passage:` prefixes, token truncation, and attention mask creation.
- Resource discovery for `multilingual-e5-small.mlpackage` plus `e5-tokenizer/tokenizer.json` and `tokenizer_config.json`.
- `scripts/convert_e5_to_coreml.py` for offline model conversion.
- `CoreMLE5EmbeddingService` in the app target using CoreML plus `swift-transformers` `Tokenizers`.
- CoreML-first app startup with fallback to `NLEmbeddingService` and then `HashingEmbeddingService`.
- App bundle packaging copies `.build/Models/e5` into `Resources/Models/e5` when local assets exist.
- Running the conversion script on the real `intfloat/multilingual-e5-small` model.
- Bundling or installing the generated `.mlpackage` and tokenizer assets on this machine.
- Manual cross-language retrieval quality validation after real model assets are installed.

Deferred:
- Reindexing a real user corpus with `coreml-e5-small-v1`.

## Asset Layout

Runtime checks these paths:

```text
~/Library/Application Support/Mneme/Models/e5/
  multilingual-e5-small.mlpackage
  e5-tokenizer/
    tokenizer.json
    tokenizer_config.json

.build/Mneme.app/Contents/Resources/Models/e5/
  multilingual-e5-small.mlpackage
  e5-tokenizer/
    tokenizer.json
    tokenizer_config.json
```

Conversion command:

```bash
uv run --with transformers --with torch --with coremltools \
  scripts/convert_e5_to_coreml.py --output-dir .build/Models/e5
```

Then rebuild the bundle:

```bash
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer scripts/build_app_bundle.sh
```

Current local assets:

```text

.build/Mneme.app                                  257M
.build/Mneme.app/Contents/Resources/Models/e5    241M
```
