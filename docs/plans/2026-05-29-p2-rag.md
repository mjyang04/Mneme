# Mneme P2 — Local RAG Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the documented "Ask my files" path with citations, preserving the offline invariant and using MLX local generation with an extractive fallback.

**Architecture:** `QueryService.answer` retrieves top-K chunks and delegates answer text generation to a `RagAnswerGenerator`. The product implementation wraps `MLXLocalRagAnswerGenerator` in `ResilientRagAnswerGenerator`; if MLX cannot load or generate, it falls back to `ExtractiveRagAnswerGenerator`.

**Tech Stack:** Swift 5.10 / macOS 14 · SwiftPM · existing `MnemeCore` search stack · SwiftUI segmented mode switch.

---

## Scope

Included:
- `RagAnswer` public model with generated text and clickable citations.
- `RagAnswerGenerator` protocol.
- `ExtractiveRagAnswerGenerator` fallback that cites every snippet it uses and says "不知道" when no evidence is available.
- `QueryService.answer(_:, topK:, filter:)` and `QueryService.answerStream(_:, topK:, filter:)`.
- Search UI mode switch: Search / Ask.
- QuickSearch prefix `?` routes to ask mode.
- `RagPromptBuilder` with numbered evidence and no-chain-of-thought instruction.
- MLX local answer generation using `Qwen3-0.6B-4bit` cached under Application Support.
- Streaming RAG answer chunks through `ChatSession.streamResponse(...)`, surfaced in Search and QuickSearch.
- `mlx.metallib` build/sign/copy path for generated `.app` bundles.

Deferred:
- Larger-model selection and context-window tuning.
